# Epic 020: First Complete Investment Committee

Status: Epic 20.2 - End-to-End Investment Committee Integration Completed

## Purpose

Run one complete AI investment committee meeting from a single typed request
without changing the frozen v2.0 architecture.

The committee remembers before it reasons. `MeetingService` remains the
application entry point, builds memory and research context first, renders the
Investment Intelligence Context once, then delegates committee execution to the
existing orchestrator and agent runtime.

## Completed Architecture

Epic 20.2 wires the Epic 20.1 product skeleton into the existing execution path:

- `InvestmentCommitteeRequest` captures the user-facing committee request.
- `MeetingService.execute_investment_committee()` runs the complete integration
  flow.
- `ContextService` assembles research and memory context.
- `InvestmentIntelligenceService` or its mock equivalent builds the unified
  intelligence context.
- `InvestmentIntelligenceRenderer` renders that context for committee prompts.
- `CommitteeMeetingOrchestrator` runs the fixed committee agent sequence.
- `AgentRuntime` renders prompts, calls the configured LLM provider, validates
  JSON schema output, and returns persistable `AgentResult` values.
- `InvestmentCommitteeReport` is assembled deterministically from the generated
  agent outputs and rendered intelligence context.

No new providers, intelligence layers, runtime patterns, or trading behavior
were introduced.

## Execution Flow

```text
InvestmentCommitteeRequest
        |
        v
MeetingService
        |
        v
ContextService builds research and memory context
        |
        v
Investment Intelligence Context Service builds provider-neutral signals
        |
        v
InvestmentIntelligenceRenderer renders prompt-ready context
        |
        v
CommitteeMeetingOrchestrator creates per-agent meeting context
        |
        v
AgentRuntime executes committee agents with prior discussion
        |
        v
MeetingService builds InvestmentCommitteeReport
```

## Runtime Responsibilities

Each committee member receives:

- the original user request fields;
- the rendered research and memory context;
- the rendered Investment Intelligence Context;
- previous agent outputs when available;
- the committee member's existing role metadata.

The runtime continues to use the existing prompt-backed agent API and schema
validated outputs.

## Report Responsibilities

The final `InvestmentCommitteeReport` is deterministic. It does not parse
free-form model text beyond loading the existing schema-validated JSON payloads
already generated by the runtime.

The report populates:

- ticker, topic, and time horizon from the typed request;
- macro, sector, risk, momentum, and sentiment views from the rendered
  Investment Intelligence Context;
- fundamental, bull case, risk, and final decision content from committee agent
  outputs;
- key risks by collecting generated committee risk lists;
- decision, confidence, and recommended action from the Chairman output.

## Demo

Developers can run a local mock-provider meeting with:

```text
python -m parakeetnest meeting "Should we add to NVDA?" --ticker NVDA
```

The existing `parakeetnest meeting` CLI command remains unchanged in shape and
continues to execute through `MeetingService`.
